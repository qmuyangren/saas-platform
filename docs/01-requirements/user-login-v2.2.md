## 详细登录流程

### 1. 账号密码登录 (支持手机号/邮箱/账号)

**业务规则**:
- 账号类型识别：
  - 手机号：`/^1[3-9]\d{9}$/`
  - 邮箱：`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
  - 其他：用户名
- 密码错误 1 次后显示图形验证码
- 连续失败 5 次，锁定账号 15 分钟
- 用户 ID 使用 UUID

---

### 2. 手机号 + 短信验证码登录

**业务规则**:
- 滑块验证：极验/易点云等第三方服务
- 短信服务商：阿里云短信/腾讯云短信
- 自动注册：生成随机密码，用户后续可设置
- 手机号必须唯一
- 用户 ID 使用 UUID

---

### 3. 邮箱 + 邮箱验证码登录

**业务规则**:
- 邮箱格式验证：标准邮箱正则
- 邮件服务商：SendGrid/阿里云邮件推送
- 自动注册：生成随机密码，用户后续可设置
- 邮箱必须唯一
- 用户 ID 使用 UUID

---

### 4. 第三方 OAuth 登录

```
┌─────────────────────────────────────────────────────────────────┐
│                    第三方登录流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  支持的第三方平台：                                             │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  平台      │ 适用场景    │ 配置要求                      │  │
│  ├──────────────────────────────────────────────────────────┤  │
│  │  微信      │ 国内 C 端     │ 微信开放平台账号             │  │
│  │  钉钉      │ 企业用户    │ 钉钉开放平台账号             │  │
│  │  GitHub    │ 开发者用户  │ GitHub OAuth App             │  │
│  │  (可扩展)  │             │                              │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  登录流程：                                                     │
│  用户点击第三方登录按钮                                         │
│           ↓                                                     │
│  跳转到第三方授权页面                                           │
│           ↓                                                     │
│  用户确认授权                                                   │
│           ↓                                                     │
│  第三方回调 (携带 code)                                         │
│           ↓                                                     │
│  使用 code 换取 OpenID + UserInfo                               │
│           ↓                                                     │
│  检查 OpenID 是否绑定账号                                        │
│    ├─ 已绑定 → 登录成功，颁发 Token                             │
│    └─ 未绑定 → 自动注册新账号 → 绑定 OpenID → 登录成功          │
│                                                                 │
│  自动注册规则：                                                 │
│  • 微信：昵称 + 随机字符 → 用户名，随机密码                     │
│  • 钉钉：钉钉昵称 + 随机字符 → 用户名                           │
│  • GitHub: GitHub login → 用户名                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5. 第三方账号绑定/解绑

```
┌─────────────────────────────────────────────────────────────────┐
│                    绑定第三方账号                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  绑定流程 (用户已登录):                                         │
│  用户进入账号设置 → 点击"绑定微信"                              │

│  跳转到微信授权页面                                             │
│           ↓                                                     │
│  用户确认授权                                                   │
│           ↓                                                     │
│  微信回调 (携带 code)                                           │
│           ↓                                                     │
│  换取 OpenID                                                    │
│           ↓                                                     │
│  检查 OpenID 是否已被其他账号绑定                                │
│    ├─ 未绑定 → 绑定成功                                         │
│    └─ 已绑定 → 提示"该微信已绑定其他账号"                        │
│                                                                 │
│  解绑流程：                                                     │
│  用户进入账号设置 → 点击"解绑微信"                              │
│           ↓                                                     │
│  验证登录密码 (安全确认)                                        │
│           ↓                                                     │
│  检查是否还有其他登录方式                                       │
│    ├─ 有其他方式 → 解绑成功                                     │

│                                                                 │
│  绑定关系表：                                                   │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  UserOAuth                                               │  │

│  │  ├── provider      VARCHAR(20)  -- wechat/dingtalk/github│  │
│  │  ├── open_id       VARCHAR(100) -- 第三方 OpenID         │  │
│  │  ├── union_id      VARCHAR(100) -- 第三方 UnionID(可选)  │  │
│  │  ├── avatar        VARCHAR(255) -- 第三方头像           │  │
│  │  ├── nickname      VARCHAR(100) -- 第三方昵称           │  │
│  │  ├── created_at    DATETIME                             │  │
│  │  └── @@unique([provider, open_id])                      │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6. 账号注册

```
┌─────────────────────────────────────────────────────────────────┐
│                    账号注册流程                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  注册方式：                                                     │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  用户名 + 密码 + 手机号                                   │  │
│  │  用户名 + 密码 + 邮箱                                     │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  注册流程：                                                     │
│  用户输入用户名                                                 │
│           ↓                                                     │
│  检查用户名是否已存在                                           │
│    ├─ 已存在 → 提示"用户名已被使用"                             │
│    └─ 未存在 → 继续                                             │
│           ↓                                                     │
│  用户输入密码                                                   │
│           ↓                                                     │
│  密码强度验证 (6-20 位，至少包含字母和数字)                      │
│           ↓                                                     │
│  用户输入手机号/邮箱                                            │
│           ↓                                                     │
│  验证格式 + 发送验证码                                          │
│           ↓                                                     │
│  输入验证码并验证                                               │
│           ↓                                                     │
│  检查手机号/邮箱是否已注册                                      │
│    ├─ 已注册 → 提示"该手机号/邮箱已注册"                        │
│    └─ 未注册 → 创建账号                                         │
│           ↓                                                     │
│  注册成功，自动登录                                             │
│                                                                 │
│  唯一性检查：                                                   │
│  • 用户名唯一                                                   │
│  • 手机号唯一                                                   │
│  • 邮箱唯一                                                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```
│                                                                 │
```

## 数据库设计 (v2.2 UUID 版本)

```prisma
// User 表
model User {
  id            String    @id @default(uuid()) @db.VarChar(36)
  phone         String?   @unique
  email         String?   @unique
  username      String?   @unique
  password      String?   // bcrypt 加密，第三方登录可为空
  avatar        String?
  nickname      String?
  status        UserStatus @default(ACTIVE)
  failedAttempts Int      @default(0)
  lockedUntil   DateTime?
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt
  
  // 关联
  oauthAccounts UserOAuth[]
  sessions      Session[]
  auditLogs     AuditLog[]
  
  @@index([phone])
  @@index([email])
  @@index([username])
}

// 第三方账号绑定表
model UserOAuth {
  id          String   @id @default(uuid()) @db.VarChar(36)
  userId      String   @db.VarChar(36)
  provider    String   // wechat/dingtalk/github
  openId      String
  unionId     String?  // 微信 unionId(可选)
  avatar      String?
  nickname    String?
  accessToken String?  // 第三方 access_token(可选)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  
  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  
  @@unique([provider, openId])
  @@index([userId])
}

// Session 表
model Session {
  id           String   @id @default(uuid()) @db.VarChar(36)
  userId       String   @db.VarChar(36)
  token        String   @unique  // JWT Token
  refreshToken String   @unique
  device       String?  // 设备信息
  ip           String?  // 登录 IP
  userAgent    String?
  expiresAt    DateTime
  createdAt    DateTime @default(now())
  
  user         User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  
  @@index([userId])
  @@index([token])
}

// 审计日志表
model AuditLog {
  id        String   @id @default(uuid()) @db.VarChar(36)
  userId    String?  @db.VarChar(36)
  action    String   // login/logout/register/bind/unbind/reset_password
  provider  String?  // password/sms/email/wechat/dingtalk/github
  status    String   // success/failure
  ip        String?
  userAgent String?
  details   Json?
  createdAt DateTime @default(now())
  
  @@index([userId])
  @@index([action])
  @@index([createdAt])
}

enum UserStatus {
```
